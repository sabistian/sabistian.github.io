Lua 语言基础

基础杂项

  • and 和 or 短路求值，一种习惯写法

无法确定 b 是否为真时候还是if - else 标准写法的好

  • 字符串连接 “..”，Lua 中的字符串是不可变的值，连接操作符只会创建一个新的字符串，不会对其原操作数进行任何修改。

  • 操作符的优先级

  • Lua 中的链表写法 虽然很少会用到 链表都用数组来实现

  • 可以在最后一个元素后面加一个逗号，使其不去特殊处理，也可以用分号来区分不同类型的元素。

  • 允许多重赋值,并且是先对等号右边的所有元素求值，然后才执行赋值。将等号右边值的个数调整到与左边变量的个数一致，少的话赋值成nil，多余的会丢掉。
    主要用法，交换两个元素，接收函数的多个返回值

  • 尽可能的使用局部变量，访问局部变量更快，并且局部变量可以垃圾收集器在其作用域结束的时候自动回收，一种习惯写法

缩短变量的作用域可以提高可读性，并不一定要在方法的起始处声明所有的局部变量

  • lua 的控制结构 if、for和while 以 end 作为结尾，repeat 以 until 作为结尾
    lua 的if-else 写法与 mysql 语句 中的一致
    repeat - until 相当于 do - while 保证至少执行一次，并且在循环体声明的局部变量可以用于条件测试

  • for 语句 两种形式，数字型 for 和 泛型

函数

正常函数的用法，需要将所有参数放到一对圆括号中，差别在于，当这个函数只有一个参数，并且此参数只有一个字面字符串或table 构造式，那么圆括号是可有可无的。

Lua 提供的冒号操作符使调用o.foo 时将 o 隐含地作为函数的第一个参数。

函数的参数与多重赋值一致，可以舍弃和用nil填充

多重返回值：字符串的模式匹配，返回起始，终止索引

lua 函数可以返回多个返回值，将函数作为一条语句舍弃所有返回值，将函数作为表达式的一部分时，只保留第一个返回值，一个函数调用是一系列表达式中的最后一个元素(或仅有一个元素)时，获得其所有返回值。
多重赋值、函数调用时传入的实参列表，table 的构造式和 return 语句

函数调用不是一系列表达式的最后一个元素，只产生一个值

跟踪某个特定的函数调用

Lua的函数定义实际就是一条赋值语句，创建了一种类型为“函数”的值，并将这个值赋予一个变量。
可以将表达式“function(x)end” 视为一种函数的构造式

像可以接受另一个函数作为实参的，为“高阶函数”。 函数在Lua 中是一种“第一类值”

闭合函数
一个函数定义在另一个函数之内，并且位于内部的函数可以访问外部函数中的局部变量
这项特征称之为“词法域”。
“词法域” 和 “第一类值” 是两项极其有用的特性
在Lua中允许访问的原因在于函数是“第一类值”

定义递归函数的问题是说

Lua 的尾调用机制不会耗费任何栈空间

想要受益于“尾调用消除”时，务必要确定当前的调用是一条“尾调用”。
判断的准则就是“一个函数在调用完另一个函数之后” ，是否就无其他事情需要做了

编译、执行、与错误

dofile 是一个辅助函数，loadfile 做真正核心的工作，加载并编译，讲编译结果作为一个函数返回
项目程序在 LuaManager 类中的Run() 方法 luaState.Dofile(“main.lua”)中执行的脚本程序

函数 loadstring 和 loadfile 类似, 不同之处它从字符串中读取代码，而非文件读取，开销较大 谨慎使用

Lua 提供的关于动态链接的功能都聚集在一个函数中，package.loadlib,有两个字符串参数：动态库的完整路径和一个函数名称。

错误 通过调用error 函数并传入一个错误消息的参数。
在发生错误时不能简单地崩溃或退出。只要发生了一个错误，Lua就应该结束当前程序块并返回应用程序

协同程序

  • 协同程序可以处于4种不同的状态：挂起，运行，死亡，和正常。
  • 在创建时，它处于挂起状态。协同程序不会在创建它时自动执行其内容，可以通过函数status来检查协同程序的状态
  • coroutine.resume 用于启动或再次启动一个协同程序的执行，并将其状态由挂起改为运行

  • Lua 中的协同程序挂起唤醒写法

  • lua 提供的是一种“非对称的协同程序”，一个用于挂起，另一个用于恢复执行

完整的实例 - 马尔可夫链算法

根据原始文本中n个单词的序列来确定后面的单词，生产随机的文本。本例中，n定位2，程序先读取原始文本，并创建一个table,table的创建过程为：以每两个单词为一个前缀，在table中创建一个列表，该列表记录了原始文本中所有位于该前缀之后的单词。
构建好这个table后，程序就利用它来生产随机文本。结果中的每个单词都来自于它在原始文本中的前缀，而具有相同前缀的单词出现在结果中的概率也与原始文本一样。最终会得到一串比较随机的文本。
